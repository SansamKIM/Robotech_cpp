# 객체지향 프로그래밍

## 1. 객체지향 이전의 프로그래밍
### 1) 비구조적 프로그래밍
  #### - 코드를 구조화하지 않고 작성
  #### - ex) 어셈블리어
### 2) 절차적 프로그래밍
  #### - 소스 코드를 여러 부분으로 나눠서 활용
    - 프로그램 전체 흐름 코드 + 프로시저 코드
  ####  - 코드의 논리구조를 모듈화
    - 재사용
    - 라이브러리같은 기능 
  ####  - ex) C, 코볼
  ####  - 한계
    - 논리적 다층 구조를 표현하는 데에 한계 (규모가 커지면 프로시저도 대폭 증가 => 프로시저 간 관계 복잡해짐 => 중복 호출)
    - 접근하지 말아야 할 프로시저에 접근 (특히 전역 변수 변경)


## 2. 객체지향 프로그래밍
### ***객체라는 논리적인 개념으로 코드를 구성하는 것***
<br>
<br>

### 절차적 프로그래밍

   프로그램이 수행할 기능을 프로시저 단위로 쪼개어 문제 해결 = 하향식 방법

### 객체지향 프로그래밍

   객체라는 논리적인 단위를 먼저 정의한 후에 조합해 프로그램 수행할 기능 구성 = 상향식 방법
<br>
<br>

### 객체지향 프로그래밍 필요성 (요구사항)
#### - 현실 모델링: 현실 세계의 사물이 가진 특징과 관계를 표현할 수 있어야 한다.
#### - 논리적 계층 관계 표현: 복잡하고 다양한 객체 간 계층 관계를 표현할 수 있어야 한다.
#### - 접근 제어: 불필요한 정보 노출과 접근을 효과적을 제거할 수 있어야 한다.
